# RAG Chat Storage Microservice

A production-ready Spring Boot microservice for storing and managing chat histories generated by a RAG (Retrieval-Augmented Generation) based chatbot system.

---

## Tech Stack

| Layer | Technology |
|-------|-----------|
| Language | Java 17 |
| Framework | Spring Boot 3.2 |
| Database | PostgreSQL 15 |
| ORM | Spring Data JPA / Hibernate |
| Auth | API Key (via `X-API-Key` header) |
| Rate Limiting | Bucket4j |
| API Docs | Springdoc OpenAPI (Swagger UI) |
| Containerization | Docker + Docker Compose |
| DB Management UI | pgAdmin 4 |
| Testing | JUnit 5 + Mockito |

## Quick Start

### Prerequisites
- Docker & Docker Compose installed

### 1. Clone and configure

```bash
git clone https://github.com/your-org/rag-chat-service.git
cd rag-chat-service

# Copy and edit environment variables
cp .env.example .env
# Edit .env and set a strong API_KEY value
```

### 2. Start all services

```bash
docker-compose up --build
```

This starts:
- **App** ‚Üí http://localhost:8080
- **pgAdmin** ‚Üí http://localhost:5050
- **PostgreSQL** ‚Üí localhost:5432

### 3. Access Swagger UI

Open: [http://localhost:8080/swagger-ui.html](http://localhost:8080/swagger-ui.html)

Click **Authorize** and enter your `API_KEY` value.

## API Reference

All endpoints require the header:
```
X-API-Key: <your-api-key>
```

### Chat Sessions

| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/api/v1/sessions` | Create a new session |
| `GET` | `/api/v1/sessions?userId=` | List all sessions (paginated) |
| `GET` | `/api/v1/sessions/favorites?userId=` | List favorite sessions |
| `GET` | `/api/v1/sessions/{sessionId}?userId=` | Get a specific session |
| `PATCH` | `/api/v1/sessions/{sessionId}/rename?userId=` | Rename a session |
| `PATCH` | `/api/v1/sessions/{sessionId}/favorite?userId=` | Toggle favorite |
| `DELETE` | `/api/v1/sessions/{sessionId}?userId=` | Delete a session (soft delete) |

### Chat Messages

| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/api/v1/sessions/{sessionId}/messages?userId=` | Add a message |
| `GET` | `/api/v1/sessions/{sessionId}/messages?userId=` | Get message history (paginated) |

### Health Check

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/actuator/health` | Service health status |

## Example Requests

### Create a Session
```bash
curl -X POST http://localhost:8080/api/v1/sessions \
  -H "X-API-Key: your-api-key" \
  -H "Content-Type: application/json" \
  -d '{"userId": "user-123", "title": "My First Chat"}'
```

### Add a Message
```bash
curl -X POST http://localhost:8080/api/v1/sessions/{sessionId}/messages?userId=user-123 \
  -H "X-API-Key: your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "sender": "USER",
    "content": "What is RAG?",
    "retrievedContext": "RAG stands for Retrieval-Augmented Generation..."
  }'
```

### Get Messages (paginated)
```bash
curl "http://localhost:8080/api/v1/sessions/{sessionId}/messages?userId=user-123&page=0&size=20" \
  -H "X-API-Key: your-api-key"
```

## Security Features

- **API Key Auth** ‚Äî every request must include `X-API-Key` header
- **Rate Limiting** ‚Äî per-IP using token bucket algorithm (Bucket4j)
- **CORS** ‚Äî configurable allowed origins
- **Soft Delete** ‚Äî sessions are never physically deleted
- **Non-root Docker user** ‚Äî container runs as unprivileged user
